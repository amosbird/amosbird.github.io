What can be done with today's keyboard layout?

2015-09-21


Talking about keyboard layout with programmers, most time you will hear the word Dvorak. I’m not a fan of this, since switching to these keyboard layouts will be rough if you have a lifetime’s experience with QWERTY. You’ll need months — maybe even as much as a year — to get back up to your current speed. When you use someone else’s computer, you’ll have to use the QWERTY layout — so all your Dvorak muscle memory will only hurt you. iPads and iPhones only support the QWERTY layout for their touch-screen keyboards, so you can’t rearrange the software keyboard’s layout to match your Dvorak layout.

So we need to improve the existence, a.k.a the QWERTY one. Not long ago I’ve posted a question on a keyboard forum. I think splitting spacebar is the most ergonomic choice we can take on QWERTY layout, since nobody types space with both hands. One more key modifier will provides a lot available shortcuts. Sadly, till now I still cannot find a laptop with such keyboard design. Also, trackpoint is a must for me. So I’m going to rearange keys on my t450s.

##Remapping key modifiers

It just feels right to make CapsLock remap to Ctrl. What seems less popular is to remap Right Alt to Ctrl as well. This is indeed ergonomic, because after pressing CapsLock a period of time, you’re going to suffer the infamous pinky problem. Using thumb is much more robust. Based on this remapping, I added some new shortcuts which takes ergonomics to the next level.

  1. Remap Ctrl ,/Ctrl . to Ctrl Tab/Ctrl Shift Tab

    In Windows, switching between tabs is quite useful. Most application provides Ctrl Tab/Ctrl Shift Tab to move forward and backward through tabs. By this remapping, you will be very agile to switch, only with right hand.

  2. Remap Ctrl [ to Esc

    This is inspired by Vim. Esc is quite far away from the home row.

  3. Set Ctrl ; to invoke Listary

    This is an advisory remapping. I’m feeling ergonomic after doing so.

  4. Let Shift Space only send Space

    Windows users, especially those with Chinese input methods, often run into “key stealing” problems. Shift Space will always toggle full-width input. If you type Upper case with Shift, this senario should be quite familiar.

  5. Remap Alt Tab/Alt Shift Tab to Alt x/Alt z

    I’m experiencing this remap about two weeks. It relieves my little finger.

If we could design a keyboard from ground up today, it should be much more satisfying. However, backward compatibility is everywhere in computer science. Whatever ditches it, it dies.
